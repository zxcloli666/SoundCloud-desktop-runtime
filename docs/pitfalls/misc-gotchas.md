# Мелкие грабли

Меньше по объёму, но каждая стоила реального времени на диагностику.

## `skia_safe::Font::default()` — 0-глифовый typeface

**Симптом**: текст монтировался, layout считался верно, но на экране
ничего не рисовалось — ни одного видимого глифа.

**Причина**: `Font::default()` даёт typeface без единого глифа — не
крашится, просто рисует пустоту.

**Фикс**: реальный шрифт только через `FontMgr::default().
legacy_make_typeface(None, FontStyle::default())` — резолвится в "Noto
Sans" (fontconfig на тестовой машине видит 708 семейств).

## esbuild `jsxDEV` + `NODE_ENV=development` — безобидный шум в консоль

**Симптом**: `console.error` про "Static children should be an array" на
каждый рендер.

**Причина**: esbuild's `jsx: 'automatic'` вместе с
`NODE_ENV=development` даёт `jsxDEV`-вызовы вместо `jsx`, и
react-reconciler's dev-режим ругается на форму статических children.

**Статус**: не блокирует рендер. Чистого фикса без потери настоящих
dev-ошибок реконсилера (которые полезны) не нашлось — оставлено как
есть, осознанно.

## Skia `color`-пропы: изначально только `[r,g,b,a]`-массивы

**Статус**: исторический — упрощение спайка 5 (`color`-пропы принимали
только массивы, не CSS-строки типа `"#fff"`/`"rgba(...)"`, которые
принимает настоящий `Skia.Color()`). Снято позже: `parse_color` в
`scene.rs` теперь полноценный CSS-парсер (hex `#rgb`/`#rrggbb`/
`#rrggbbaa`, `rgb()`/`rgba()`, `transparent`, именованные цвета) — массивы
по-прежнему тоже работают. Упомянуто здесь как пример: если видите
"известное упрощение" где-то в коде/доках — проверьте, не снято ли оно
уже, прежде чем повторно на него натыкаться.

## `repeat` resize mode для `<Image>` — недокументированный пробел

**Статус**: `resizeMode="repeat"` (react-native.tsx's `<Image>`) молча
деградирует до `cover` вместо тайлинга. `@sc/ui` никогда его не
использует, поэтому не проявилось ни в одном тесте — если понадобится
консumerу этого движка, потребует реальной реализации в
`scene.rs::draw_resized_image`.
