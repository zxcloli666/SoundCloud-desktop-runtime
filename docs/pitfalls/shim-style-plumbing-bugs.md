# Стилевой пайплайн JS↔Rust: три тихих бага

Все три найдены аудитом покрытия шимов против реального `@sc/ui` usage (не
просто грепа импортируемых компонентов, а по факту рендера/пропов) —
объединяет их то, что все три отказывали МОЛЧА: дерево монтировалось,
просто без части визуала, никакого исключения/краша.

## Array-form `style` никогда не флэттенился

**Симптом**: реальные `@sc/ui`-компоненты рендерились как пустые/
безымянные боксы — стили просто не применялись.

**Причина**: `hostConfig.ts`'s `applyStyle` сериализовал `style={[a, b]}`
как есть в JSON; Rust-сторона (`StyleInput`, untagged enum) отвергала
массив целиком. Реальный `@sc/ui` почти everywhere передаёт `style`
массивом (`[baseStyle, conditionalStyle]`), не голым объектом — этот
паттерн просто не встречался до аудита.

**Фикс**: `resolveStyle` в `hostConfig.ts` — флэттенит массивы (включая
вложенные) + резолвит функции-элементы массива — ДО `JSON.stringify`.

## `createAnimatedComponent` спреил style-массив как объект

**Симптом**: `Card`/`Button` (оба оборачивают `Pressable` через
`Animated.createAnimatedComponent`) рендерились без стилей вообще.

**Причина**: спред `[styleA, styleB]` в объект давал мусорные числовые
ключи (`{0: styleA, 1: styleB}`) — `resolveStyle` получал объект, который
не был ни валидным стилем, ни массивом, который умел разбирать.

**Фикс**: пропускать массив как есть через `createAnimatedComponent`,
пусть `resolveStyle` разбирает его uniformly (тот же путь, что и
array-form fix выше — один источник правды для "как выглядит style").

## `Record<string, unknown>` wildcard ломал контекстную типизацию TS

**Симптом**: `onLayout={(e) => ...}` тихо получал `e: any` вместо
реального типа события — TypeScript не подсказывал ни одного поля,
опечатки в обработчике не ловились компилятором.

**Причина**: wildcard-индекс `Record<string, unknown>` в базовых типах
пропов (`Props`/`PressableProps`/`TextProps`) — подтверждено
изолированным репродюсом ВНЕ кодовой базы (минимальный `.ts`-файл с той
же формой типа воспроизводил проблему сам по себе, без зависимостей).

**Фикс**: убрать wildcard, дать явные поля в каждом типе пропов.
Побочный эффект — с реальной типизацией под компилятором TS сразу
подсветил ЕЩЁ 3 скрытые дыры, которые wildcard маскировал: `Image`'s
`resizeMode` никогда не читался, `ScrollView`'s реальный prop-surface
(`contentContainerStyle` и т.д.) был не полон, `Pressable.onPress`'s
callback вообще не имел типа события. Все три закрыты тем же проходом.
